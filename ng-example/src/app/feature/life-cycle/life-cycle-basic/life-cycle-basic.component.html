https://angular.io/guide/lifecycle-hooks
  1. constructor()
  2. ngOnChanges() - Called before ngOnInit() (if the component has bound inputs) and whenever one or more data-bound input properties change. Note that if your component has no inputs or you use it without providing any inputs, the framework will not call ngOnChanges()
  3. ngOnInit() - Called once, after the first ngOnChanges(). ngOnInit() is still called even when ngOnChanges() is not fired
  4. ngDoCheck() - Called immediately after ngOnChanges() on every change detection run, and immediately after ngOnInit() on the first run
  5. ngAfterContentInit() - Called once after the first ngDoCheck()
  6. ngAfterContentChecked() - Called after ngAfterContentInit() and every subsequent ngDoCheck()
  7. ngAfterViewInit() - Called once after the first ngAfterContentChecked()
  8. ngAfterViewChecked() - Called after the ngAfterViewInit() and every subsequent ngAfterContentChecked()
  9. ngOnDestroy() - Called immediately before Angular destroys the directive or component
  • Bold entries are called only once
  • ngDoCheck() is a good indicator for change detection run
